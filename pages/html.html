HTML ​

iframe标签 ​

iframe用来创建另外一个文档的框架（引入另外一个文档）。

  • src属性：引入文档的地址。
  • width属性：设置文档框架的宽度。
  • height属性：设置文档框架的高度。
  • frameborder属性：规定是否显示边框。1显示边框，0不显示边框。
  • name属性：规定这个iframe窗口的名字，可以配合a标签的target属性使用。

补充：a的target的其他的值：target的值除了_self（自己窗口打开）、_blank（新窗口打开）、自定义的窗口名、_parent（在父级窗口打开）、_top（清除所有框架在最顶层打开）。

_parent找父窗口

_top找顶层窗口

表单 ​

表单用来向服务器端进行传值。

  • <form>标签，告诉浏览器这个双标签中包含的是内容是表单的内容。

    • method属性，提交数据的方法。默认的时候是get方法提交也就是查询字符串方式提交。还可以使用post方法提交数据。
    • action属性，将数据提交到哪里，地址可以是绝对路径也可以是相对路径。

  • label标签：为input标签定义标注。

    定义标注需要for属性，for属性的值需要和input标签的id属性的值进行绑定才行。

    html
    <label for="passwd1">输入密码：</label><input type="password" name="mima1" value="abc" id="passwd1"/>
    <label for="passwd1">输入密码：</label><input type="password" name="mima1" value="abc" id="passwd1"/>
    1

    input的id属性的值应该和label的for的值相同，这个时候完成绑定。

  • input标签：用来定义各种各样的输入框。

    通过input标签的type属性定义输入框的类型（根据type属性的值来配合使用其他的哪个属性）。

    type属性的值：

    • text，表示为一个文本输入框，让用户进行输入的。

      • name属性：该文本框传递到服务器上的标识。

      • value属性：用来定义默认值。

      • maxlength属性：表示最大输入字符的宽度。

      • disabled属性：用来禁用这个输入框。disabled="disabled"可以直接写为disabled

        布尔属性（真，true、假false）当你只写了一个属性名不指定属性值或任意的值的时候都会当成真，如果想让其为假那么就不使用该属性。

      • readonly，表示只读。它也是一个布尔属性。

        readonly和disabled都不能更改值，他们的区别就在于如果是readonly的输入框那么不能修改里面的内容但是会随着表单数据的提交再次提交，如果是disabled的输入框那么不能更改里面的内容并且表单数据提交它也不会被提交。

    • password，用来表示一个密码框。

      • name属性：给这个密码框提供一个名字，传递到服务器上的标识。
      • value属性：用来标识默认值（现在不用）。

文档中的每个元素都会生成一个矩形框，我们称为元素框。这个框用来描述元素在文档布局中所占的空间（布局）。

  • 内容区：用来放置内容。
    • 宽度：用来定义内容区的宽度。
    • 高度：用来定义内容区的高度。
  • 内边距：用来定义内容区和边框的距离。
  • 边框：边框是围绕在内容区或内边距周围的一条或多条线。
  • 外边距：定义元素和元素之间的距离。在元素周围添加的额外的空白。

宽度 ​

元素的宽度指的是内容区左边界到右边界的距离。

格式：width:value（块状元素和行内元素）

  • auto，默认值。自动计算。没有固定的宽度值。

    对于块状元素来说，宽度默认是父元素的内容区的宽度。

  • px，像素。

  • %，百分比，按照百分比来设置宽度。相对于包含他的块状元素的宽度来设置。

高度 ​

元素的高度指的是内容区上边界到下边界的距离。

  • auto，默认值。没有固定的高度。

    块状元素的高度如果为auto和块状元素中line-height的值一样。

  • px，像素。

  • %，百分比，按照百分比来设置高度。相对于包含他的块状元素的高度来设置。

    body的高度靠里面内容的行高，html靠body的高度。

    如果要撑到页面的视口高度先把html设置为100%（页面视口高度），body再设置为100%（html的高度）

宽度、高度的最大值和最小值 ​

  • 元素的内容区可以使用min-width和min-height定义最小值。

    苹果10块钱一斤，陶哥看人家长的好看给100块钱一斤。只要大于等于10就可以。

  • 元素的内容区可以使用max-width和max-height定义最大值。

    生活费3000，最多只能给3000，小于等于就可以。

男人要学会藏私房钱。

内边距 ​

任何元素都可以使用内边距，元素默认的时候没有内边距（默认的时候内边距为0）

内边距使用padding属性：

  • px，像素。

    padding的值不能为负数。

  • %，使用百分比来设置内边距。水平（左右）还是垂直（上下）都相对于父级元素内容区的宽度来计算。

简写格式：（一个值、两个值、三个值、四个值 ）

  • padding:top/right/bottom/left（一个值）
  • padding:top/bottom left/right(两个值)
  • padding:top left/right bottom（三个值）
  • padding:top right bottom left（四个值）

单边内边距：

  • padding-top
  • padding-right
  • padding-bottom
  • padding-left

注意： 行内元素上设置左右内边距可以正常使用，但是上下内边距不会改变行内元素的位置。

边框 ​

border:1px solid green;

边框有三个组成：宽度（border-width），样式（border-style）、颜色（border-color）

  • 宽度，具体的边框宽度默认由于浏览器决定一般是2px或3px。border-width。
  • 样式，默认为none，所以边框即使有默认的宽度也看不到，如果样式为none边框默认为0。
  • 颜色，默认的边框颜色是元素自身的文本颜色。 border-color

边框样式 ​

样式设置使用border-style。

值：

  • none，没有样式。默认就是none。
  • solid，实线。
  • 其他值，自行百度。

简写格式：（一个值、两个值、三个值、四个值 ）

  • border-style:top/right/bottom/left（一个值）
  • border-style:top/bottom left/right(两个值)
  • border-style:top left/right bottom（三个值）
  • border-style:top right bottom left（四个值）

单边样式：

  • border-top-style
  • border-right-style
  • border-bottom-style
  • border-left-style

边框宽度 ​

边框宽度：border-width，值为px。

简写格式：（一个值、两个值、三个值、四个值 ）

  • border-width:top/right/bottom/left（一个值）
  • border-width:top/bottom left/right(两个值)
  • border-width:top left/right bottom（三个值）
  • border-width:top right bottom left（四个值）

单边样式：

  • border-top-width
  • border-right-width
  • border-bottom-width
  • border-left-width

边框颜色 ​

简写格式：（一个值、两个值、三个值、四个值 ）

  • border-color:top/right/bottom/left（一个值）
  • border-color:top/bottom left/right(两个值)
  • border-color:top left/right bottom（三个值）
  • border-color:top right bottom left（四个值）

单边样式：

  • border-top-color
  • border-right-color
  • border-bottom-color
  • border-left-color

边框没有是没有，有边框没有颜色是没有颜色（可以将边框颜色设置为transparent）

行内元素的边框左右可以生效但是上下不会改变元素的位置。

简写属性 ​

格式：border-top:width style color

格式：border-right:width style color

格式：border-bottom:width style color

格式：border-left:width style color

整个边框 ​

格式：border:width style color

水平居中 ​

  • 文本、行内元素、行内块状元素居中：text-align:center;

垂直居中 ​

  • 单行文本垂直居中：line-height=height

定位 ​

定位是指将元素移动到哪里，允许元素相对于某个__基准__来__移动__。

确定基准使用position属性，怎么移动使用left、top、right、bottom。

偏移属性 ​

  • top，指定了定位元素上边界相对于基准上边界的偏移位置。（正下，负上）
  • bottom，指定了定位元素下边界相对于基准下边界的偏移位置。（正上，负下）
  • left，指定了定位元素左边界相对于基准左边界的偏移位置。（正右，负左）
  • right，指定了定位元素右边界相对于基准右边界的偏移位置。（正左，负右）

定位基准 ​

格式：position:value

  • static，默认值，正常生成元素框，元素不能移动。

  • relative，元素可以进行偏移，元素的形状所占空间也与正常情况下相同，它的基准是元素本身应该在位置。

    注意：

    1. 元素相对定位之后，即使该元素移动走，本来的位置也不会被其他元素占据。
    2. left\right同时使用left生效，top\bottom同时使用top生效。
    3. relative时移动的百分比相对于包含的父元素计算，不是自身。

  • absolute，元素完全脱离文档流，元素在文档中所占据的空间没有了。其基准相对于__容纳块__来确定。

    • 绝对定位的容纳块是position属性的值不是static的最近的祖辈元素。

      也就是说相对于最近的一个定位了的祖辈元素移动。

      要零花钱（我要消费要移动）----》 爸爸（position的值是不是static，是static就没有钱）---》爷爷（position的值是不是static，是static就没有钱）---》。。。。。。。

      一辈一辈往上找，找到了position的值不等于static的就用这一辈作为定位基准了。

    • 如果没有祖辈元素的position值不为static，元素的容纳块就是初始容纳块。

      初始化容纳块是根元素（html）的容纳块。它是由浏览器自己建立的，是窗口大小的矩形，我们可以理解为页面的第一屏。

    注意：

    1. 定位之后生成的是个块级元素。

    2. 如果绝对定位中的元素浮动，将不会造成决定定位元素的高度塌陷。

    3. 当在移动的时候坐标的基准是按照容纳块的内边距的外边界来定位的。

    4. 公式：

      容纳块宽度+容纳块padding宽度 = left + margin-left + border-left + padding-left + width + padding-right + border-right + margin-right +right

      • 同时使用left、right来定位，并且width为auto，他们都会自动计算。

      • 同时使用top、bottom来定位，并且height为auto，他们都会自动计算。

      • 当margin为auto，宽度、高度、偏移属性都设置了，这个时候margin自动计算（平分）

Web Worker ​

Web Worker 是什么 ​

  1. Web Worker 是 HTML5 标准的一部分，这一规范定义了一套 API，允许我们在 js 主线程之外开辟新的 Worker 线程，并将一段 js 脚本运行其中，它赋予了开发者利用 js 操作多线程的能力。

  2. 因为是独立的线程，Worker 线程与 js 主线程能够同时运行，互不阻塞。所以，在我们有大量运算任务时，可以把运算任务交给 Worker 线程去处理，当 Worker 线程计算完成，再把结果返回给 js 主线程。这样，js 主线程只用专注处理业务逻辑，不用耗费过多时间去处理大量复杂计算，从而减少了阻塞时间，也提高了运行效率，页面流畅度和用户体验自然而然也提高了。

Web Worker能干什么 ​

  1. 虽然 Worker 线程是在浏览器环境中被唤起，但是它与当前页面窗口运行在不同的全局上下文中，我们常用的顶层对象 window，以及 parent 对象在 Worker 线程上下文中是不可用的。另外，在 Worker 线程上下文中，操作 DOM 的行为也是不可行的，document对象也不存在。但是，location和navigator对象可以以可读方式访问。除此之外，绝大多数 Window 对象上的方法和属性，都被共享到 Worker 上下文全局对象 WorkerGlobalScope 中。同样，Worker 线程上下文也存在一个顶级对象 self。

详细信息请参考：Functions and classes available to Web Workers
