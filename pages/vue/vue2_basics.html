Vue2 ​

Vue介绍 ​

  • Vue 是一套用于构建用户界面的渐进式框架。作者尤雨溪，其知乎、github

    • 所谓的构建用户界面核心就一句话：用数据来驱动视图！data => UI
    • 渐进式：用什么引进什么，用vueRouter引入vueRouter，用vuex引入vuex

  • Vue框架有配套的成熟全家桶（Vue/VueRouter/Vuex）, 类似于react全家桶（React/react-router/redux）同时也非常容易与第三方库UI组件库（ElementUI）一起使用。

  • Vue框架除了开发web应用（H5、PC），还可以借助一些工具如 uniapp来开发手机App、微信小程序等其他端应用。

    官网地址： https://cn.vuejs.org/

为什么要使用框架开发 ​

  • 提高项目开发效率。在一个企业中，一个项目基本是不可能从0到1开发完成的，中间的开发过程都是借助一些框架、工具来快速完成，这些工具俗称轮子。在企业中，“ 时间就是金钱，效率就是生命 ”； 提高自己就业机会。 企业招聘要求必须会Vue或React。

Vue核心特点 ​

  1. 语法简洁、体积小
  2. 数据驱动视图，底层通过diff算法来高效更新页面，不需要去手动操作真实DOM，避免造成页面性能上面的问题。
  3. 单向、双向数据绑定
  4. Vue采用组件化开发方式，便于分治(分开治理)。
  5. 声明式编码，让开发人员无需直接操作DOM，提高开发效率

认识mvvm设计模式 ​

vue中，采用了一种mvvm设计模式，来控制页面的更新。

  • MVVM是 Model-View-ViewModel的简写，是前端视图层的概念，主要关注于视图层分离，
    • M:Model 数据层,
    • V:View 视图层
    • VM:ViewModel (控制层，在view和model之间起调度作用)

defineProperty实现数据代理 ​

  • 在new Vue时候data对象中设置的属性都会被设置到vm实例中的_data的属性中。
  • 再通过defineProperty把_data对象中的属性全部都代理到vm身上，这样做的目的是便于通过this进行访问。

核心代码如下：

javascript
let _data = {
        age: 10,
        sex: '男',
    }
    
	let vm = {};
    // 将data中的属性代理到了vm对象中,便于vue中通过this访问
    for (let key in _data) {
        Object.defineProperty(vm, key, {
            // 属性的setter
            set(newVal) {
                _data[key] = newVal;
            },
            // 属性的getter
            get() {
                return _data[key];
            }
        })
    }

    console.log(vm.age)； // 触发getter拦截
    vm.age = 100; // 触发setter拦截

小结数据劫持和代理 ​

  • 无论是数据劫持和代理，底层API都是通过Object.defineProperty实现
  • 之所以可以通过this.xx = 新值来更新页面。因为已经把_data中的属性代理到了vm身上。
  • 对数据进行修改时，会触发属性的setter劫持操作，从而新数据响应式的更新到页面。

Object.defineProperty的局限性 ​

Object.defineProperty存在的问题：

  • 无法拦截对象的新增和删除属性操作。当对响应式对象新增属性、删除属性，页面不会更新。
  • 无法拦截数组。即直接通过下标修改数组，界面也不会自动更新。

key内部原理总结 ​

  1. 虚拟DOM中的key作用

    为了唯一标识虚拟DOM节点，便于底层进行diff算法对比。更加高效更新页面。

    vue会根据 更新后的新数据 生成新的虚拟DOM,在与旧的虚拟DOM进行对比，比较规则如下：

  2. 对比规则：

    (1). 新虚拟DOM要找到与旧虚拟DOM相同的key:

    • key一样，在继续比较其内层元素，如果节点和内容一样，则复用原来已经在页面中渲染好的节点，若不要则重新生成新节点。
    • key不一样，直接生成新的节点并渲染页面中

  3. 用index作为key可能会导致的问题：

若对数据进行逆序添加、删除等破坏顺序操作，会导致数据混乱，尤其结构中还有输入类的节点如复选框、文本框等。

  1. 开发中如何选择key？：
    1. 最好使用每条数据的唯一标识作为key，比如id、身份证号、学号等唯一值。
    2. 如果不存在对数据的进行逆序添加、删除等破坏顺序操作。仅用于渲染列表展示。这时使用index作为key是没有问题的。

是否有没有复用旧节点，看页面元素有没有闪烁。没有闪烁就是旧地复用

watch和computed区别 ​

  • computed具有依赖缓存效果，依赖没变化不会重新计算。其主要处理数据依赖相关的问题。
  • computed 能完成的功能，watch都可以完成。
  • watch能完成的，computed不一定能完成。如watch 支持异步操作（副作用），computed不支持异步操作。

computed本来就是用来返回计算后的值得，是需要马上得到的，而不是要等一段时间才能得到的,且computed函数中return是同步执行的，数据直接返回给页面显示，是没有办法拿到异步请求结果的。

双向绑定原理 ​

原理：v-model 在内部为不同的输入元素使用不同的 property 并监听不同的事件：从而实现双向绑定

  • text 和 textarea 元素使用 value property 和 input 事件；
  • checkbox 和 radio 使用 checked property 和 change 事件；
  • select 字段将 value 作为 prop 并将 change 作为事件。

自定义指令 ​

Vue除了有v-bind、v-html、v-if、v-show等指令外，还允许我们自定义指令。

自定义指令作用：对 DOM 元素进行底层操作 。

指令分类：

  • 全局指令。 Vue.directive(指令名称，指令对象)

  • 局部指令。对象中添加directives属性

一个指令定义对象可以提供如下几个钩子函数 (均为可选)：

  • bind：只调用一次，指令第一次绑定到元素时调用。在这里可以进行css初始化设置。
  • inserted：元素节点插入到文档之后
  • update：所在的节点更新时调用。
  • componentUpdated：指令所在组件的 VNode 及其子 VNode 全部更新后调用。
  • unbind：只调用一次，指令与元素解绑时调用。一般在这里做些事件相关的解绑等操作。

生命周期 ​

生命周期，顾名思义，就是从生到死的过程。 而生命周期钩子，就是从生到死过程中的关键节点。 普通人的一生有哪些生命周期钩子（函数）呢？

  1. 出生
  2. 上学
  3. 第一份工作
  4. 买房
  5. 结婚
  6. 生子
  7. 退休
  8. 临终遗言

上面的每个关键节点，我们总会做些相应的事情。

什么是Vue生命周期

从Vue实例初始化创建、运行、到销毁期间，总是伴随着各种各样的事件，我们称之为生命周期函数或生命周期钩子。

生命周期函数分类： 主要分为以下几个阶段

  1. 创建阶段
  2. 挂载阶段
  3. 更新阶段
  4. 销毁阶段

  • 创建阶段：

    • beforeCreate（创建前）： 此时，vm实例还没创建好，也没有初始化好 data 和 methods 属性，即不能通过this来访问，一般在这里给Vue原型设置一些初始化的属性或消息订阅。
    • created（创建后）：此时，vm实例已经创建好了，data 和 methods 已经初始化完毕，可以进行调用。一般在这里可以通过data数据初始化ajax请求

  • 挂载阶段

    • beforeMount：此时已经完成了模板的编译，但是还没有挂载到页面中
    • mounted（挂载后）：此时，已经将编译好的模板，挂载到了页面指定的容器中显示。可以进行真实的DOM操作。一般初始化ajax请求，绑定自定义事件、订阅消息等初始化操作。

  • 更新阶段：

    • beforeUpdate（更新前）：在数据更新之前调用。这里的更新之前是指数据已经变化，但页面还未更新
    • updated（更新后）：在数据更新之后调用。页面已经更新

  • 销毁阶段：

    • beforeDestroy（销毁前）：实例销毁之前调用。这里一般对组件身上进行解绑事件、清除定时器、取消订阅等收尾操作。注意：销毁后自定义事件会失效，但原生DOM事件依然有效，但是已不具备响应式能力。

    • destroyed（销毁后）：销毁后调用。

初始化ajax请求写在created中还是mounted中呢？

答案：两者都可以！如果是不依赖页面dom的值这时可以放到created中，若要依赖页面DOM的值则写在mounted中是最稳妥的。

重点掌握以下几个生命周期函数：

  1. created(创建后)：初始化网络请求,或对某个事件处理函数进行防抖或节流包装
  2. mouted（挂载后）: 做一些DOM操作（要避免），初始化ajax请求，绑定自定义事件、开启定时器、订阅消息等初始化操作。
  3. beforeDestroyed(销毁前): 对组件进行销毁。其身上某些功能也要销毁。 如：解绑事件、清除定时器、取消订阅等操作。

注意事项：生命周期的函数不能写成箭头函数，其this不是指向VM实例，而是window

Vue中操作DOM ​

在Vue中，虽然不建议直接操作真实DOM，但有些情况不可避免的需要操作DOM元素。

在Vue中常用两种方式来操作

  • 通过[refs]来引用DOM元素（单独使用可能会有问题，因为vue数据更新是异步）
  • 通过[$nextTick]来操作

refs ​

示例代码：

html
<template>
  <div>
    <p id="info" ref="pEle">你好，{{ message }} - {{info}}</p>
    <button @click="getContent">获取p标签的值</button>
  </div>
</template>

<script>
export default {
  name: 'App',
  data() {
    return {
      message: '尚硅谷',
      info: '你好'
    }
  },
  methods: {
    getContent() {
      
      this.message = '尚硅谷YYDS1'
      this.message = '尚硅谷YYDS2'
      this.info = '大家好才是真的好'
      
      // 方式一：
      console.log(document.getElementById('info').innerText) // 不建议(同步没问题，异步会有问题)
      // 方式二：
      console.log(this.$refs.pEle.innerText) //  不建议(同步没问题，异步会有问题)

      // 方式三：
      this.$nextTick(() => {
        // Vue中，由于页面DOM更新是个异步的过程，会把所有数据更新放到一个任务队列中。

        console.log('nextTick')
        console.log(this.$refs.pEle.innerText)
        console.log(document.getElementById('info').innerText)
      })
    }
  },
};
</script>

  • nextTick回调会等待所有的数据更新到真实DOM中，才会执行。
  • 在Vue中操作DOM最佳实践：在nextTick中通过ref去操作真实DOM，这是最稳妥的！！！

最佳实践：refs结合nextTick案例 ​

由于Vue是异步更新DOM的，所以的数据更新都会放入异步更新队列中。

要想获取更新完毕之后的最新DOM，一定要在nextTick回调中进行操作！！！

nextTick会等待所有DOM节点更新完毕后在执行，所以获取到时最新的DOM。

课堂案例：动态添加li节点，在获取li子节点个数。

html
<template>
  <div>
    <ul ref="ulEle" id="ulEle">
      <li v-for="item in users">{{ item }}</li>
    </ul>
    <button @click="addUser">添加</button>
  </div>
</template>

<script>
export default {
  name: 'App',
  data() {
    return {
      users:['张学友','刘德华','郭富城'],
    }
  },
  methods: {
    addUser(){
      this.users.push('汪哥') // 数据更新是异步
      
      console.log( document.getElementById('ulEle').children ) // 3 都是更新之前的
      console.log( this.$refs.ulEle.children ) // 3 都是更新之前的

      // 要在数据异步更新完毕只有才可以获取DOM li节点的个数
      this.$nextTick(()=>{
        console.log('nextTick');
        console.log( this.$refs.ulEle.children ) // 4 最新的DOM值
      })
    }
  },
};
</script>
